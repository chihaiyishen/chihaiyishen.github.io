三 知乎日报 API 分析

此项目的内容都是来源于知乎日报 API 上获取的，获取最新文章的接口：http://news-at.zhihu.com/api/4/news/latest

返回的数据为 JSON 格式的

{
      "date": "20170501",
      "stories": [
    {
      "images": [
        "https://pic3.zhimg.com/v2-a2c638494e994fd327e6f8438c987a66.jpg"
      ],
      "type": 0,
      "id": 9392864,
      "ga_prefix": "050117",
      "title": "卖得那么多，vivo 不就是广告营销做得好吗？"
    },
    {
      "images": [
        "https://pic3.zhimg.com/v2-0c643ed85c3ba99464e1442375892bc2.jpg"
      ],
      "type": 0,
      "id": 9392165,
      "ga_prefix": "050115",
      "title": "学（曾保持了二十年的统治地位的）C++ ，要先想清楚这些"
    },
      ]
}

其中，date 代表当日时间，stories 为一个 JSON 数组，包含了一组文章数据，如 title（文章标题），images（文章配图），id（文章 ID），API 详细分析可以点击这里进行查看 知乎日报 API 分析

依靠得到的文章 ID，再加上获取文章详细内容的接口：http://news-at.zhihu.com/api/4/news/9392864，就可以获取数据。
即为 html 格式的字符串，文章详细页的界面就是用 WebView 来呈现的，WebView 是最简单且呈现效果最好的方式，其中还包含了 CSS 文件的地址： “css”: [“http://news-at.zhihu.com/css/news_qa.auto.css?v=4b3e3“] 这个需要下载到本地（assets）然后导入工程中，因为 CSS 文件是连续使用的且一般不会频繁更改的，所以可以直接作为本地资源。

DrawerLayot

滑动菜单可以说是 Material Design 中最常见的效果之一了。虽说这个功能看上去好像挺复杂的，不过借助谷歌提供的各种工具，我们可以很轻松地实现非常炫酷的滑动菜单效果。

所谓的滑动菜单就是将一些菜单选项隐藏起来，而不是放置在主屏幕上，然后可以通过滑动的方式将菜单显示出来。这种方式既节省了屏幕空间，又实现了非常好的动画效果，借助 DrawLayout 控件，可以轻松实现滑动菜单。

CoordinatorLayout

CoordinatorLayout 可以说是一个加强版的 FrameLayout，这个布局是由 Design Spport 库提供的。它在普通情况下的作用和 FrameLayout 基本一致，不过既然是 Design Support 库中提供的布局，那么就必然有一些 Material Design 的魔力了。

CoordinatorLayout 可以监听其所有子控件的各种事件，然后自动帮助我们做出最为合理的响应。

AppBarLayout

AppBarLayout 实际上是一个垂直方向的 LinearLayout，它在内部做了很多滚动事件的封装，并应用了一些 Material Design 的设计理念。

当 AppBarLayout 接收到的滚动事件的时候，它内部的子控件是可以指定如何去影响这些事件的，通过 app:layout_srcollFlags 属性就能实现。这个属性的值有 scroll|enterAlways|snap，其中，scroll 表示当布局中有 RecyclerView 向上滚动的时候，Toolbar 会跟着一起向上滚动并实现隐藏；enterAlways 表示当 RecyclerView 向下滚动的时候，Toolbar 会跟着一起向下滚动并重新显示；snap 表示当 Toolbar 还没有完全隐藏或显示的时候，会根据当前滚动的距离，自动选择是隐藏还是显示。

Toolbar

Toolbar 是乎有点陌生，但对于它的另一个相关控件 ActionBar，应该就比较熟悉。每个活动最顶部的那个标题栏其实就是 ActionBar。不过 ActionBar 由于其设计的原因，被限定只能位于活动的顶部，从而不能实现一些 Material Design 的效果，因此官方现在已经不再建议使用 ActonBar 了。

Toolbar 的强大之处在于，它不仅继承了 ActionBar 的所有功能，而且灵活性很高，可以配合其他控件来完成一些 Material Design 的效果。

SwipeRefreshLayout

SwipeRefreshLayout 用于实现下拉刷新功能的核心类，它是由 support-v4 库提供的。我们把想要实现下拉刷新功能的控件放置到 SwipeRefreshLayout 中，就可以迅速让这个控件支持下拉刷新。在使用此控件的布局中，调用 setColorSchemeResources() 方法来设置下拉刷新进度条的颜色，调用 setOnRefreshListener() 方法来设置一个下拉刷新的监听器，当触发了下拉刷新操作的时候就会调用这个监听器的 onRefresh() 方法，然后我们在这里去处理具体的刷新逻辑就可以了。

FloatingActionButton

FloationgActionButton 是 Design Support 库中提供的一个控件，这个控件可以帮助我们比较轻松地实现悬浮按钮的效果。其中可以通过，android:layout_gravity=”bottom|end” 属性指定将这个控件放置于屏幕的位置，其中 end 的工作原理和 start 一样，即如果系统语言是从左往右的，那么 end 就表示在右边，如果系统语言是从右往左的，那么 end 就表示在左边。既然是悬浮，那么就理所应当会有投影，这里通过 app:elevation 属性来为 FloatingActionButton 指定一个高度值，高度值越大，投影范围也越大，

NavigationView

NavigationView 是 Design Support 库中提供的一个控件，它不仅是严格按照 Material Design 的要求设计的，而且还可以将滑动菜单页面的实现变得非常简单。

使用 NavigationView 之前，我们需要准备好两个东西：menu 和 headerLayout。menu 是用来在 NavigationView 中显示具体的菜单项的，headerLayout 则是用来在 NavigationView 中显示头部布局的。

文章展示界面（activity_daily_content.xml）:

在文章展示界面中借助 CollapsingToolbarLayout 实现可折叠式标题栏的效果。

CollapsingToolbarLayout 是一个作用于 Toolbar 基础之上的布局，它也是由 Design Support 库提供的。CollapsingToolbarLayout 可以让 Toolbar 的效果变得更加丰富，不仅仅是展示一个标题栏，而是能够实现非常华丽的效果。

不过，CollapsingToolbarLayout 是不能独立存在的，它在设计的时候就被限定只能作为 AppBarLayout 的直接子布局来使用。而 AppBarLayout 又必须是 CoordinatorLayout 的子布局。

此控件中 app:contentScrim 属性用于指导 CollapsingToolbarLayout 在趋于折叠状态以及折叠之后的背景色。app:layout_scrollFlags 属性之前就有使用过，只不过之前是给 Toolbar 指定的，现在也移到外面来了。其中，scroll 表示 CollapsingToolbarLayout 会随着内容详情的滚动一起滚动，exitUntilCollapsed 表示当 CollapsingToolbarLayout 随着滚动完成折叠之后就保留在界面上，不在移除屏幕。

数据接收

此项目是通过 知乎日报 API 接收数据，其中涉及网络操作，网络操作封装成类于 util 包中。

HttpUtil 类

在 HttpUtil 类中封装静态 sendOkHttpRequest() 方法。此方法接收两个参数：String address，Callback callback。其中第一个参数表示访问的 url ，第二个参数通过 Callback ，进行回调来处理访问结果。

public class HttpUtil {

    public static void sendOkHttpRequest(String address, Callback callback){
        OkHttpClient client = new OkHttpClient();
        Request request = new Request.Builder().url(address).build();
        client.newCall(request).enqueue(callback);
    }

}

Utility 类

在 Utility 类中封装了静态 handleZhihuDailyResponse() 方法。此方法接收两个参数：String response, List dailyDataList。其中第一个参数是服务器返回的数据，第二个参数是 List 数组类用来存放接收的数据。DailyData 是继承 DataSupport 的数据库类。

public static boolean handleZhiHuDailyResponse(String response, List<DailyData> dailyDataList) {
    try {
        JSONObject jsonObject = new JSONObject(response);
        JSONArray jsonArray = jsonObject.getJSONArray("stories");
        for (int i = 0; i < jsonArray.length();i++) {
            Gson gson = new Gson();
            Daily daily = gson.fromJson(jsonArray.getString(i), Daily.class);
            DailyData dailyData = new DailyData();
            dailyData.setId(daily.getId());
            dailyData.setTitle(daily.getTitle());
            dailyData.setImages(daily.getImages().get(0));
        }
        return true;
    } catch (JSONException e) {
        e.printStackTrace();
    }
    return false;
}

主界面数据展示

在主界面中使用 RecyclerView 来展示从知乎日报 API 中接收的数据，使用自定义的 DailyAdapter 适配器。DailyAdapter 通过 queryDaily()，requestDaily() 方法初始化，

queryDaily() 方法

private void queryDaily() {
      if (dailyList.size() > 0) {
          dailyAdapter = new DailyAdapter(dailyList);
          recyclerView.setAdapter(dailyAdapter);
      }
      else
      {
          requestDaily();
      }

  }

requestDaily() 方法

private void requestDaily() {
       showProgressDialog();
       String dailyUrl = " http://news-at.zhihu.com/api/4/news/latest";
       HttpUtil.sendOkHttpRequest(dailyUrl, new Callback() {
           @Override
           public void onFailure(Call call, IOException e) {
               closeProgressDialog();
               Toast.makeText(MyApplication.getContext(), "加载失败", Toast.LENGTH_SHORT).show();
           }

           @Override
           public void onResponse(Call call, Response response) throws IOException {
               String reponseText = response.body().string();
               boolean reslut = false;
               reslut = Utility.handleZhiHuDailyResponse(reponseText,dailyList);
               if (reslut) {
                   MainActivity.this.runOnUiThread(new Runnable() {
                       @Override
                       public void run() {
                           closeProgressDialog();
                           queryDaily();
                       }
                   });
               }
           }
       });

   }

文章内容展示

showDailyContent() 方法

private void showDailyContent(DailyContent dailyContent) {
        String title = dailyContent.getTitle();
        String imageId = dailyContent.getImage();
        Log.d("gjg", "ShowDailyContent : " + imageId);
        String css = "<link rel=\"stylesheet\" href=\"file:///android_asset/src.css\" type=\"text/css\">";
        String html = "<html><head>" + css + "</head><body>" + dailyContent.getBody() + "</body></html>";
        html = html.replace("<div class=\"img-place-holder\">", "");

        ActionBar actionBar = getSupportActionBar();

五 总结

自此项目的主要内容都以讲解完毕，通过此项目把之前学习的知识使用一遍，加深印象。在此项目中遇到的最大的问题就是对数据的处理，最初的时候对于访问到的 JSON 数据处理不是很好，展示的内容并不能按自己的意愿来展示，WebView 中的加载总是有问题，这部分花费了很多的时间。在此之间通过查看别人写的代码，查找相关的资料，逐渐解决问题。但此项目还有很大不足：主界面的数据还不能刷新，每次访问数据都要从网络上获取，没有做缓存，滑动菜单目前只有日报可以点击，其它都没有去实现。但随着技能的增长，会逐步完善。长路漫漫，唯 Android 作伴。项目完整代码
